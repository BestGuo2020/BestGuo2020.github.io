公开的代码仓库真的可以随意提交吗？

2021-04-29- 2022-05-25
- -
小赫赫的项目日志-Git

直接给出结果，不行！！！

Access deined: You do not have permission push to this repository
fatal: Cou1d not read from remote repository.

Please make sure you have the correct access rights
and the repository exists.

起因

今天上午和舍友进行 git 的多人合作测试的时候，基本的项目配置以及用户名和邮箱都已经配置好，最后再次进行代码提交测试的时候提示权限不足的问题。

嗯？

尝试了很多次依然不行！

我的疑问

不是公开仓库就可以随便提交的吗？之前听说那些开源的项目不就是所有人都可以修改的吗？所有人都可以进行提交代码到远程仓库的吗？

当然不是，因为这是错觉。😊😊

经过

其实之前在刚刚创建完成 gitee 账号，将代码提交的远程仓库的时候。看到我自己建的代码仓库有这一项。也就是成员管理的那一块。

开始并没有太在意这个成员管理。现在发现成员管理真的是非常之重要，只有远程代码仓库中有这个成员才可以提交。这样就可以保证代码不会被成员以外的无关人员进行恶意提交。

不仅仅是 gitee 才有，github 上面也有邀请成员的功能。看到那个绿色的按钮了吗？

私有仓库就不用说了，共有仓库也是不能随便提交，要想提交，除非你是该仓库的一员。

结果

最后的结果，当然就是将用户添加到仓库成员就可以进行愉快的提交咯。
