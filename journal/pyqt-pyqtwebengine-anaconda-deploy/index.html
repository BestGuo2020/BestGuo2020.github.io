银河麒麟arm版v10系统打包pyqt程序

2023-09-18- 2023-12-30
- -
编程常见问题记录-AnacondaPythonPyQtPyQtWebEngine

项目背景

最近公司要做一个桌面端应用，并且该应用要在 win10，win7 和 银河麒麟 v10 银河麒麟 arm 版 v10系统中，这里主要细说 银河麒麟 arm 版 v10 操作系统，使用 nuitka 打包 python 程序至可执行文件。

该操作系统虽然自带 python3.8 和 Qt 组件，但是没有 QtWebEngine 这个组件，自己尝试用 pip 安装 PyQtWebEngine，但是卡在 qmake 一直安装失败，因此最后我们使用 anaconda 来打包这个项目。

开始打包

我的 anaconda 虚拟环境是 python 3.10，安装的依赖有以下

conda install -c conda-forge openpyxl pyqt5-sip pyqtwebengine requests jieba

然后使用 pip 安装 nutika

pip install Nuitka

开始打包

nuitka3 --standalone --show-progress --show-memory --enable-plugin=pyqt5,glfw,dll-files --include-package-data=package_name=data --windows-icon-from-ico=./icon/pc.ico --static-libpython=no --output-dir=out/release1.0.87_kylin main_ui.py

项目打包完成后，由于 Nuitka 打包不会很完整，直接运行会报错。

解决办法：复制 resource，translation到PyQt5/Qt5 目录下

huawei@huawei-L410-KLVU-WDU0:~/xxxx/out/release1.0.87_kylin/main_ui.dist$ cp -rf ./translations/ ./PyQt5/Qt5/
huawei@huawei-L410-KLVU-WDU0:~/xxxx/out/release1.0.87_kylin/main_ui.dist$ cp -rf ./resources/ ./PyQt5/Qt5/

这样子，一个带PyQtWebEngine的项目就可以启动了。

但是，还有一种情况，在不是Python3.9的项目中就会提示以下错误，这个问题是由PyQtWebEngine引起的。

报错 5：No suitable graphics backend found

anaconda 指定的 python3.9 虚拟环境不会出现问题

No suitable graphics backend found

解决办法：

# Linux 平台暂时使用软件OpenGL渲染
if sys.platform == 'linux':
  QCoreApplication.setAttribute(Qt.AA_UseSoftwareOpenGL)

在窗口打开之前，加入以上代码，项目就可正常启动，设置这个参数的目的在Linux上使用软件OpenGL渲染。

运行闪退

闪退后报 NSS 错误： NSS error code: -8023

[411062:419788:0917/235959.246870:ERROR:nss_util.cc(166)] Error initializing NSS with a persistent database (sql:/home/huawei/.pki/nssdb):
[411062:419788:0917/235959.247066:ERROR:nss_util.cc(124)] Error initializing NSS without a persistent database: NSS error code: -8023
[411062:419788:0917/235959.247083:FATAL:nss_util.cc(126)] nss_error=-8023, os_error=0

追踪与中断点陷阱 (核心已转储)

解决办法：这个我参考了一篇Qt的博客。

Qt添加QWebEngine模块后打包在无Qt的环境上运行出错的问题 - 一笔一世界 - 博客园 (cnblogs.com)

他是直接复制nss文件夹到他的项目中，不过我们要复制nss文件夹里的文件到我们的项目中。

cp /usr/lib/aarch64-linux-gnu/nss/* .

这样的话，就不会报错闪退了。
